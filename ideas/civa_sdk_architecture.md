# Архитектура интеграции игр (CIVA SDK)

Чтобы все фичи платформы (созвоны, лобби, уведомления, друзья) работали в **любой** игре (и уже существующих, и будущих), нам нужен единый стандарт. Поскольку сейчас при заходе в игру Хаб выгружается из памяти браузера (переход по ссылке), игра остается один на один с пользователем.

Решение, которое используют Steam (Steamworks), Discord (GameSDK) и VK Play — это **SDK (Software Development Kit)**.

## Главная концепция: `@civa/sdk`
Мы создадим небольшую библиотеку (`civa-sdk.js`), которую каждая игра должна будет подключать у себя (через npm или тег `<script>`). Этот SDK берет на себя всю платформенную часть, чтобы авторам игр не нужно было писать это с нуля.

### Как это решит задачи платформы:

#### 1. Групповой активный созвон ➔ Лобби
- **В Хабе:** Вы сидите в голосовом чате (Группа).
- **Хост:** Нажимает "Играть". Хаб передает в SDK игры ID вашей группы.
- **В Игре (SDK):** Игра загружается и говорит: `civa.matchmaking.hostLobby({ groupId: 123 })`.
- **У остальных (в Хабе или других играх):** SDK получает по вебсокету сигнал и показывает уведомление: *"S1mple создал лобби. [Зайти]"*. При клике их перекидывает в игру сразу в нужное лобби.

#### 2. Контекстное меню по видео/аватарке игрока
- Игра вообще не должна знать, кто у кого в друзьях. Она знает только `userId` игроков в матче.
- При клике ПКМ по аватарке в игре, автор игры вызывает всего одну функцию:
  ```javascript
  civa.ui.showContextMenu({ 
    userId: 'user_456', 
    x: event.clientX, 
    y: event.clientY 
  });
  ```
- SDK сам поверх игры рисует наше фирменное черное меню ("Добавить в друзья", "Написать", "Пожаловаться").

#### 3. Уведомления поверх любой игры
- SDK автоматически создает поверх игры невидимый слой (`Shadow DOM` с `z-index: 999999`).
- SDK сам держит WebSocket соединение с главным сервером CIVA.
- Если тебе прислали сообщение в ЛС, SDK сам нарисует красивую синюю карточку в углу экрана, даже если ты внутри игры (например, Свояка). Автор игры для этого вообще ничего не делает.

## Пошаговый план перехода к единообразию

### Шаг 1: Выделение `civa-sdk` (Пакет)
Мы создаем в нашем монорепозитории пакет `packages/civa-sdk`. 
Туда мы переносим логику:
- Соединения с вебсокетами (уведомления, чаты).
- Рендеринг `ToastContainer` (уведомлений) на чистом JS/React, который можно инжектить куда угодно.
- Рендеринг глобального меню (Shift+Tab Оверлей).

### Шаг 2: Интеграция в первую игру (например, Свояк)
Мы заходим в репозиторий Свояка и:
1. Удаляем оттуда его собственную систему авторизации (теперь он просто берет токен из `civa.auth.getToken()`).
2. Добавляем вызов `civa.init('game_svoyak')` при старте.
3. Проверяем, что во время игры в Свояк приходят уведомления о сообщениях из Хаба.

### Шаг 3: Shift+Tab Оверлей
Раз уж SDK теперь живет внутри игры, мы можем добавить туда перехватчик клавиш `Shift + Tab`. При нажатии игра затемняется, и поверх открывается полноценный интерфейс CIVA (Чаты, список друзей, ачивки), не выходя из игры.
